# SEO brief, opportunity estimation, readiness & confidence

All numeric outputs are labelled **(est.)**. Any input that is unknown is
**`Verification Required`** — never fill it with a guess. Titles, meta, H1, and
slugs are expert copy recommendations grounded in the *verified* keyword and
product; keyword lists, competitor counts, and "questions people ask" must come
from real searches.

## Action Plan fields (Step 7)

- **Priority** — `Urgent` (verified launch ≤ ~14 days out, or opportunity window
  closing), `High`, `Medium`, or `Watch`. Drive from verified freshness +
  window_days + competition, not the report's tier alone.
- **Why review it** — one sentence of evidence (demand + monetization + gap).
- **Best publish date** — for a verified upcoming launch: ~1–2 days before
  launch. For evergreen: ASAP if the SERP gap is open, else next content slot.
  Unknown launch date ⇒ `Verification Required` before setting.
- **Estimated traffic opportunity (est.)** — from the primary keyword's demand
  and SERP gap: `High` (clear volume + weak page 1), `Medium`, `Low`. Base it on
  verified search interest; if search volume is unknown ⇒ `Verification
  Required`.
- **Estimated affiliate opportunity (est.)** — verified commission × price ×
  recurring, expressed as a per-sale value and a monthly range. Unknown
  commission or price ⇒ `Verification Required` (do not use "40% assumed").
- **SEO difficulty (est.)** — rubric below.
- **Suggested article length** — by intent/difficulty: comparison/"best" listicle
  2,000–3,500w; single-product review 1,500–2,500w; how-to/tutorial 1,200–2,000w;
  news/launch first-look 800–1,200w.
- **Suggested review deadline** — Urgent: within 3 days; High: within 1 week;
  Medium: within 2 weeks; Evergreen: schedule within the opportunity window.

### SEO difficulty rubric (est.)
Score from the **verified** SERP, not the report:
- **Low** — page 1 has ≤2 authority/review results; mostly forums/thin pages.
- **Medium** — 3–6 authority results; some optimized reviews.
- **High** — 7+ authority reviews, or G2/Capterra/big-media dominating page 1.
Adjust up if the brand keyword is owned by the vendor's own pages; down if SERP
is stale (old dates).

## SEO Research Brief fields (Step 8)

- **Primary keyword** — the highest-intent head term (usually `<product> review`
  or `<product> alternatives`/`<category> software`), chosen from real SERP data.
- **Secondary keywords** — 3–6 supporting terms (pricing, vs-competitor,
  features, login, discount) that appear in autosuggest/related searches.
- **Long-tail keywords** — 4–8 specific, lower-competition phrases (questions,
  use-cases, "for <audience>", "is <product> worth it").
- **Search intent** — informational / commercial-investigation / transactional
  (state which and what the page must do to satisfy it).
- **Competitor count** — real count of ranking reviews on page 1 (YouTube +
  authority sites). Unknown ⇒ `Verification Required`.
- **Questions people ask** — from People-Also-Ask / forums / Reddit (real). ⇒
  `Verification Required` if not gathered.
- **Suggested title** — ≤ ~60 chars, primary keyword front-loaded, year if
  evergreen (e.g. "<Product> Review (2026): …").
- **Suggested meta description** — ≤ ~155 chars, includes primary keyword + a
  differentiator + a soft CTA.
- **Suggested H1** — matches intent; not identical to the title tag.
- **Suggested URL slug** — short, hyphenated, keyword-first (e.g.
  `product-name-review`).

## Review-readiness test (Step 9)

Return **`READY TO REVIEW`** only if ALL are verified/present:
- launch date verified (or credibly "evergreen, active"),
- affiliate program verified active + commission known,
- official website + current pricing verified,
- competition/SERP counted,
- primary keyword + search intent set,
- freshness confirmed fresh/moderate (or unknown-but-verified demand).

Otherwise **`NEEDS MORE RESEARCH`** — list exactly which of the above are
`Verification Required`. Missing affiliate terms or missing keyword/SERP data
are the most common blockers.

## Confidence Score (Step 10)

`Confidence = round(100 × verified_inputs / total_key_inputs)` over these key
inputs: launch date, affiliate program, commission, price, official website,
product status, competition/SERP count, freshness signal, primary keyword,
search demand. Count an input only if `verified` (report value corroborated or
independently established); `Verification Required`/`conflicts`-unresolved do not
count. Report the score as a %, and name the 1–2 inputs most limiting it.

Guidance: ≥80% strong (act now); 50–79% usable with the noted gaps; <50%
treat as a lead, not a recommendation — resolve blockers first.
